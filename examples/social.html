How It Works

This example shows the classic "friend-of-friend" problem that's perfect for graph databases. The graph connects:

  • You - The current user
  • Direct Friends - Alice and Bob (your immediate connections)
  • Friend-of-Friends - Carol, Dave, Eve (people you might know)

The query finds people who are 2 hops away - friends of your friends that you're not yet connected to. This is the basis for "People You May Know" features in social networks.

The Query

// Find friend-of-friend suggestions
const suggestions = await db
  .nodes('User', { id: 'me' })
  .follow('FRIENDS_WITH')
  .follow('FRIENDS_WITH')
  .except(myFriends)
  .groupBy('id')
  .orderBy('count', 'desc')
  .exec();

// Results show mutual friend count
console.log(suggestions);
// [
//   { id: 'dave', name: 'Dave', mutualFriends: 2 },
//   { id: 'carol', name: 'Carol', mutualFriends: 1 },
//   { id: 'eve', name: 'Eve', mutualFriends: 1 }
// ]

Advanced Social Queries

// Find influential users (most connections)
const influencers = await db
  .nodes('User')
  .withCount('FRIENDS_WITH')
  .orderBy('count', 'desc')
  .limit(10)
  .exec();

// Find shortest path between two users
const path = await db
  .shortestPath(
    { type: 'User', id: userId1 },
    { type: 'User', id: userId2 },
    { relationship: 'FRIENDS_WITH' }
  )
  .exec();

// Find common interests with suggestions
const withInterests = await db
  .nodes('User', { id: 'me' })
  .follow('LIKES')
  .follow('-LIKES')
  .where({ id: { not: 'me' } })
  .groupBy('id')
  .orderBy('count', 'desc')
  .limit(20)
  .exec();

// Build personalized feed
const feed = await db
  .nodes('User', { id: 'me' })
  .follow('FRIENDS_WITH')
  .follow('POSTED')
  .orderBy('created_at', 'desc')
  .limit(50)
  .exec();
