How It Works

This example shows how a job board can use graph relationships to match candidates with relevant opportunities. The graph connects:

  • User - The job seeker (you)
  • Skills - Technologies and competencies (JavaScript, React)
  • Jobs - Open positions requiring specific skills
  • Companies - Organizations posting jobs

When you click "Run Query", watch as the query traverses from your profile through your skills to find matching remote jobs at various companies.

The Query

app.ts
// Find remote jobs matching my skills
const jobs = await db
  .nodes('User', { id: 'me' })
  .follow('HAS_SKILL')
  .follow('-REQUIRES_SKILL')
  .where({ remote: true })
  .exec();

// Results include job details and company info
console.log(jobs);
// [
//   { id: 'frontend-dev', title: 'Frontend Developer', remote: true, company: 'Startup Inc' },
//   { id: 'fullstack-dev', title: 'Full-Stack Developer', remote: true, company: 'Tech Corp' }
// ]

Advanced Queries

advanced.ts
// Find jobs at companies where my contacts work
const referralJobs = await db
  .nodes('User', { id: 'me' })
  .follow('KNOWS')
  .follow('WORKS_AT')
  .follow('-AT_COMPANY')
  .where({ status: 'open' })
  .exec();

// Find jobs similar to my current role
const similarJobs = await db
  .nodes('Job', { id: myCurrentJobId })
  .follow('REQUIRES_SKILL')
  .follow('-REQUIRES_SKILL')
  .where({ id: { not: myCurrentJobId } })
  .groupBy('id')
  .orderBy('count', 'desc')
  .limit(10)
  .exec();
